计算机网络

IP寻址

互联网中数据的传输依赖于路由器中的路由控制表，本文旨在探寻这张表的形成与使用

静态路由

定义：由管理员手动定义网络路由信息，并将该信息输入到路由器中。

缺点：工作量巨大，路由表巨臃肿，加入新的网络时每个路由器中的信息都要添加。同时当路由器发生故障时，整个网络会瘫痪，需要管理员手工恢复。

动态路由

由管理员设置好路由协议，相邻的网络之间会交流自己知道的网络连接信息。

路由控制范围

1. 自治系统（路由控制域）：一般是一个公司、组织机构、学校等不希望暴露给外界的小型单位，自治系统中的路由协议采用内部网关协议，如RIP或者OSPF。

2. 互联网：所有的自治系统组成的网络，路由协议采用外部网关协议，如BGP。

路由算法

1. 距离向量算法：每个路由器中维护一张表格，记录了到达目标网络的下一跳路由地址（如果有多个选择，则选择跳数最少的），RIP就是基于距离向量算法的协议。

2. 链路状态算法：每个路由器中维护当前自治系统的网络拓扑结构，所以每个路由器内的信息都是一致的。显然，保持所有路由器节点的信息一致需要大量内存和CPU资源的支持。OSPF就是基于链路状态算法的协议。

路由协议

RIP（路由信息协议）

实现：每个路由器定期向全网（LAN）广播自己的路由控制信息（30s），如果没有收到路由控制信息，就断开链接（考虑到网络丢包的情况，一般会在6次收不到信息后断开链接）。每个路由器根据自己收到的路由控制信息，根据距离向量算法计算出到达某个目标网络最短的跳数后，形成路由控制表。

缺点：网络数量越多，要传递的路由控制信息越大。当整个网络稳定后，还是要定期交换路由控制信息，一定程度上浪费了网络带宽。

问题：当某个网络断开链接后，虽然该网络的路由不再发送路由控制信息（以期望数据传送绕开该网络），但是它可能会收到自己曾经发送出去的信息并将跳数加一，这种情况称为“无限计数”。

解决：

  1. 规定整个网络中的最大跳数为16。
  2. 水平分割：每个路由器不再给发送给自己信息的路由器发送信息。

尚存问题：当整个网络中本身就存在环路的情况下，反向回路会成为迂回通道不断循环往复的转发信息。

解决：

  1. 毒性逆转：当链路被断开后，不是不发送消息，而是发送一个跳数为16的消息。
  2. 触发更新：一旦网络断开，立即发送消息，不再等待30s。

即使有了这些措施，但是面对复杂的网络结构，路由信息想要达到稳定的结构还是需要花费一段时间的，OSPF能较好的解决这个问题。

OSPF（开放最短路径优先）

相邻路由器之间传递链路状态生成网络拓扑信息，在自治区域内传递拓扑信息，生成路由控制表。OSPF给每一条链路赋予一个权重，选择一个权重最小的路由作为最终路由（与RIP不同）。

5种包类型

类型 包名 功能
1 问候（HELLO） 确认相邻路由器、确定指定路由器
2 数据库描述（Database Description） 链路状态数据库的摘要信息
3 链路状态请求（Link State Request） 请求从链路状态数据库获取链路状态信息
4 链路状态更新（Link State Update） 更新链路状态数据库中的链路状态信息
5 链路状态确认应答（Link State Acknowledge） 链路状态信息更新的确认应答

工作流程：

  1. 通过发送HELLO包确认是否连接。
  2. 为了同步路由控制信息，利用数据库描述包相互发送路由摘要信息和版本信息。
  3. 如果版本较老，则发送链路状态请求包请求路由控制信息
  4. 由链路状态更新包接收路由状态信息
  5. 通过链路状态确认包通知大家本地已经接收到路由控制信息。

通过这样的方式，可以在网络稳定的时候减少网络流量，并且也能达到迅速更新路由信息的目的。

工作原理：

LAN中10s发送一个HELLO包，如果没有收到，则进行连接是否断开的判断（连续四次空等则认为连接断开），在进行链接恢复操作时，路由器会发送一个链路状态更新包，通知其他路由器网络状态的变化。

链路状态更新包分为两种：网络LSA和路由器LSA，分别是以网络为中心和以路由器为中心生成的信息。

在收到这些信息后，每个路由器就都能生成一个可以表示网络结构的链路状态数据库，根据这个数据库，采用Dijkstra算法，就能生成相应的路由控制表。

问题：网络规模增大时，算法所需的时间处理时间会越来越长，对CPU和内存的要求也会越来越高。

解决：将自治系统分层，分成多个区域。要求这些区域中必须有一个主干区域与每个区域相连。由此也带来了自治系统中路由器的分类：

  1. 区域边界路由器：连接区域和主干区域的路由器
  2. 内部路由器：区域内部的路由器
  3. 主干路由器：只与主干区域内连接的路由器
  4. AS边界路由器：与外部相连接的路由器

每个区域内的路由器存放本区域的网络拓扑的数据库，在该信息的基础上计算出路由控制表，有效减少了路由控制信息，减轻处理负担。

BGP（边界网关协议）

BGP是连接不同AS的一种协议，其最终路由控制表由网络地址和下一站的路由器组表示，会根据所要经过的AS个数进行路由控制。

AS号：每个自治系统都有一个16比特的AS编号，BGP根据这个编号进行相应的路由控制。

BGP扬声器：根据BGP交换路由控制信息的路由器。

工作过程：

BGP数据包送达目标网络时，会生成一个中途经过所有AS的编号列表，这个表格就是AS路径信息访问列表（AS Path List），当出现多条路径时，BGP会从列表中选择一个较短的路由。

BGP是一个路径向量协议，在AS路径信息访问列表中包含了转发方向和距离，还有途径所有AS的编号。这种路径向量型能够检测出环路，避免了无限计数的问题，令网络更容易进入一个稳定的状态。

路由控制是跨越整个互联网的分布式系统，互联网中的路由控制，以网络中的所有路由器都持有正确的路由信息为基础，使这些路由器的信息保持准确的协议就是路由协议。路由协议散布在互联网的各个角落，是支撑互联网正常运行的一个巨大的分布式系统。
