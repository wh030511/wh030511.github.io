SSM框架

Spring

JavaEE框架，核心是IOC和AOP

IOC

将创建对象的权力转移给了框架（工厂），不能确定框架（工厂）返回的对象是自己想要的，称为控制反转。削减了应用和资源之间的耦合关系。

引入

在使用new关键字时，会造成类与类之间的耦合。实际开发中，应当尽可能的将编译时的依赖延后到运行时。一个行之有效的策略就是通过反射创建对象，将对类的依赖转为对字符串的依赖。工厂模式应运而生。

多数情况下，我们只需要一个对象即可，无需每次访问都创建一个新的对象，造成无谓的性能消耗。所以可以使用单例模式的思想，同一个对象只创建一个即可，然后将其放在一个Map中，不妨称之为对象池。

Spring中的IOC

获取IOC核心容器（可以理解为Map）

1. 两个核心容器接口：

  • ApplicationContext：采用立即加载的思想，读取配置文件的同时构建核心容器，适用于单例对象。
  • BeanFactory：采用延迟加载的策略，使用时才会构建相关对象，适用于多例对象。

tips：Spring框架可以根据xml文件中对象是单例还是多例只能决定加载策略。单例对象会将其放入到核心容器中，而多例不会

2. 三种常用实现类：

  • ClassPathXmlApplicationContext：加载类路径下的xml，使用较多。
  • FileSystemXmlApplicationContext：加载磁盘任意路径下的xml（考虑访问权限）。
  • AnnotationConfigApplicationContext：读取注解创建容器。
Spring对bean的管理细节

1. 创建bean的三种方式

  • <bean id = "key" , class = "全限定类名">，使用全限定类名指定的类的无参构造器实例，如果没有会报错。依赖注入可以解决该问题。

  • 使用某个类中的方法创建对象并放入容器中

  • 使用某个类中的静态方法创建对象并放入容器中

    <bean id = "key" , class = "相关类全类名" ， factory-method = "类中方法">

2. bean的作用范围

作用范围 说明
singleton 默认值，单例

3. bean对象的生命周期

Spring的依赖注入

个人理解：三种创建实例方式中，无论是哪种，Spring创建对象实例时都需要该对象提供无参的构造器，如果没有则会报错，这种情况下就需要一种机制，在Spring调用有参构造器时将参数赋予初值，依赖注入就是这样的机制。

另一些场景中，有些数据是不会改变的，这些数据也应该在对象创建时就确定下来，依赖注入也能完成这样的功能。

1. 能注入的数据种类

  • 基本数据类型和String
  • 其他bean类型
  • 集合类型

2. 注入方式

  • 构造函数注入

    使用constructor-arg标签注入,其中有五个属性

    • type：选取指定所有符合该数据类型的元素
    • index：选取指定索引位置的元素
    • name：选取指定名称的元素，使用较多
    • value：给选定的元素赋值
    • ref：当选定的元素依赖其他bean时使用ref替代value

    弊端：某些不需要赋值的元素也必须赋值。

  • set方法注入

    当相关类中提供了set方法时，可以使用该方式按需注入数据,解决了构造器注入的弊端。在bean标签中使用property标签即可，property标签支持三种属性

    • name：选取指定的元素，将原set方法的方法名中的set去掉，将首字母改成小写即为name的值
    • value：给选定的元素赋值
    • ref：当原定元素依赖其他bean时用ref替代value

    tips：相同的数据形式可以相互转换，如list、array和set ，map和props等。

  • 注解注入

    注解 作用
    @Component 生成该注解修饰的类的实例对象，并放在IOC容器中
    @Controller 与@Component作用相同，修饰表现层
    @Service 与@Component作用相同，修饰业务层
    @Repository 与@Component作用相同，修饰持久层
    @Autowired 该注解修饰的变量会自动注入（从IOC容器中按照数据类型、变量名的顺序依次匹配）
    @Qualifier 除了用于修饰parameter，不能单独使用。被修饰的变量会在类型注入的基础上进行名称注入。名称由value属性指定
    @Resource 该注解修饰的变量会自动注入（通过name属性指定bean的id，在IOC容器中寻找）
    @Value 自动注入基本数据类型和String，其注入值通过spEL（${表达式}）指定
    @Scope 指定bean的作用范围，通过value属性指定
Spring的其他注解

当使用注解开发时，需要用到以下注解来替换xml文件。

  • @Configuration：该注解修饰的类为配置类。
  • @Bean：该注解修饰的方法会将返回值作为bean对象放在IOC容器中，key默认为方法名，可以通过name属性指定，
  • @ComponentScan：通过该注解的value属性指定创建容器时需要扫描的包
  • @Import：通过value属性导入其他的配置类，作为当前配置类的子类
  • @PropertySource：通过value属性指定properties文件的路径
Spring和JUnit整合

JUnit没有main方法能执行的原因是，JUnit默认构造了一个main方法，会扫描该类中由@Test注解修饰的方法，去执行他们。但是这个main方法不能识别spring框架，所以解决思路就是使用spring提供的main方法替换junit默认的main方法，这就是Spring和JUnit的整合。

  1. pom文件中导入spring-test坐标

  2. 用@RunWith（SpringJUnit4Cla’s’sRunner.class）来替换JUint测试类默认的main方法

  3. 通过@ContextConfiguration（）告知spring运行器，IOC容器创建是基于注解还是xml文件的，并告知相关位置。可选的属性值包括：

    • locations：指定xml文件所在位置，classpath：+xml文件的类路径
    • classes：指定注解类所在位置，接受一个字节码文件（xxx.class）

    tips：spring版本为5.x时，相关的junit版本要大于4.1.2
