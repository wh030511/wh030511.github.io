技术对比

参加了阿里的面试，问到了多种技术之间的对比，所以写一篇博客持续更新市面上常用技术的对比。

面向对象和面向过程

严格来说并不是技术，而是不同编程语言的特点。莉莉丝笔试也问到了面向对象编程和函数式编程的区别，现整理如下。

面向对象着眼于完成需求的主体，而面向过程着眼于如何完成需求本身。这是我认为的他们最大的不同点。

而面向对象编程是如何将目光从完成需求转移到完成任务的主体上来的呢？就是通过对象了，每个对象都是一个主体，在这个主体中提供了许许多多的方法，这些方法就可以看成这个主体的各种能力。很显然，这些主体在很多场景下是可以复用的，所以面向对象天然的就比面向过程拥有更好的复用性。

不仅如此，面向对象中还引入了继承的概念，通过继承，使得一个类拥有其父类的所有功能，并且还能扩展出自己独有的能力，这样通用的需求有通用的类去完成，在通用需求上的一些特有更新就可以通过继承来与之一一匹配，而无需重新编写一个与原类重合度很高的新类，大大节省了代码量。需求是会随时更新的，通过继承的设计使得我们无需频繁的打开原有代码修改，只需要扩展出新类即可，很好的维护了开闭原则，大大减少了bug的可能。

面向对象的封装特点，我认为是为了更好的组织代码。类和对象的目的是为了向外界提供用于完成需求的各种方法，这些方法是抽象的，而在类中被封装的变量和方法最终都会被开放的方法所使用。换句话说，这些封装的成员是工具，封装将这些底层实现的细节隐藏，只想外界暴露值得外界看到的方法。这也是为什么成员变量通常都是用private修饰的原因，数据总是没有任何功能的。

至于面向对象的第三个特点多态，他是依赖于继承和方法重写的。其外在表现是声明为父类，实现是子类。这样在调用父类和子类共有的方法时，JVM通过动态绑定的机制去执行子类重写的方法。这么做的好处是可以提高代码的可维护性，当我们要修改想要使用的具体实现时，只需要更改实现的相关代码，而真正执行的代码无需更改。对于这个优势，在使用Spring依赖注入时可以更加明显的体会到。

至于函数式编程，了解的不多，只知道他的核心思想是消除变量，有一定了解后会更新。

前后端数据交互

即前后端都遵从的一种数据组织的形式。

1. json

轻量级的交互方式，易于解析，可以直接被服务端使用，便于维护。

2. xml

重量级的交互方式，解析时花费时间长，而且不同的浏览器之间解析方式不同，故需要编写许多重复代码。其组织也比较复杂，各种子节点父节点，解析困难。

其有点在于格式统一，可以与其他系统远程交互。

NoSQL数据库

在内存中存放数据，防止高并发影响数据库性能，减少数据库压力，提高查询速度。

Redis Memcached
数据结构 5种（String,list,hash,set,zset） 1种(String)
持久化 RDB、AOF 不支持
内存管理 申请内存，数据超量时触发swap，将冷数据刷新到磁盘 预分配池管理，适合作为缓存，存储速度更快
数据一致性 串行操作，无需考虑数据一致性 通过cas保证数据一致性
线程与IO 单线程，只能通过多开redis解决；需要执行排序和聚合等功能，IO较慢 多线程，可能出现锁冲突；IO多路复用，更快一些。
其他 集成了事务、复制、lua脚本等功能，支持主从，天然支持高可用集群 需要自己实现一致性hash的负载均衡算法才能解决集群

HTTP和HTTPS

HTTP：基于发送方和接收方的，无连接的，应用层协议，一般基于TCP/IP传输数据。设计的初衷是为了提供一种发布和接收HTML页面的方法。

版本 产生时间 重要内容 现状
HTTP/0.9 1991 不涉及数据包传输，规定客户端和服务器之间通信格式，只能GET请求 没有正式的作为标准
HTTP/1.0 1996 传输内容格式不限制 正式作为标准
HTTP/1.1 1997 连接复用，不是1.0时每个资源都需要建立连接 2015年前使用最广泛
HTTP/2 2015 多路复用，一个连接可以同时传输多个资源，提高速度 逐渐覆盖市场

为了解决HTTP是无状态的，起到服务端记录客户端状态的能力，提供了会话技术（Cookie和Session）和HTTP/1.1持久连接，请求行中Connection：keep-alive即保持连接（字节面试问到）。

HTTPS：基于HTTP协议，使用SSL/TLS提供数据加密处理，验证对方身份并且保护数据完整性。

底层原理：使用混合加密技术，即对称加密和非对称加密的结合。客户端使用对称加密的密钥对传输数据加密，然后使用非对称加密的公钥对密钥进行加密，这样网络上传输的就是加密后的数据和密钥，即使被截取，没有非对称加密的私钥，则无法获取真正的数据。

公钥保存再服务端的数字证书中，而证书由受信任的数字证书颁发机构CA，在验证服务器身份后颁发，证书中包含了一个密钥对（公钥和私钥）和所有者识别信息。起到了身份验证的作用。

注：https缓存可以通过Cache-Control：public来将缓存写入本地。
