工程实战

Git

Git是一个分布式版本管理工具，基于文件系统快照的方式保存不同版本的非重复数据。

一、文件目录说明

安装git后可以在创建的工作目录中使用git init命令初始化，会生成一个本地库的相关子目录和文件。为了区分不同开发人员，需要设置签名，通过git config命令指定用户名和邮箱。

二、常用命令整理

命令 含义
git status 查看工作区、缓存区状态
git add [file name] 从工作区添加到暂存区
git commit -m “commit message” [file name] 从暂存区提交到本地区
git log
git log –pretty=oneline、git log –oneline
git reflog		 查看历史记录
git reset –hard [索引值] 版本后退与前进
git diff [索引值] [文件名] 将工作区中的文件与暂存区/历史版本进行比较

注：可以使用^或~来实现版本更换。但是只能用于后退，使用规则如下

git reset参数辨析

参数 含义
soft 在本地库移动HEAD指针
mixed 除了移动指针，也会重置暂存区
hard 移动指针、重置暂存区和工作区

三、分支管理

Ⅰ、分支的好处

同时并行的推进多个功能开发，提高开发效率

各个分支在开发过程中相互独立

Ⅱ、分支操作
命令 含义
git branch -v 查看分支
git branch [分支名] 新建分支
git checkout [分支名] 切换分支
git merge [分支名] 合并分支

如果两个版本对于同一行有不同的意见，则会产生合并冲突。需要交流后决定如何采用， 解决冲突。

解决冲突的步骤如下：

  1. 编辑文件，删除特殊符号
  2. 将文件修改到合适的版本
  3. git add [文件名]
  4. git commit -m “message”

四、本地库和远程库交互

Ⅰ、基于github搭建远程库

Create a new repository

Ⅱ、本地库推送到远程库
命令 含义
git remote -v 查看本地库保存的远程库别名和地址
git remote add ‘name’ url 给远程库地址取别名
git push [别名] [分支] 将某分支推送到远程库

注：如果不是对最新版本的修改，不能推送，必须先拉取最新版进行修改后再推送。

Ⅲ、远程库到本地库
命令 含义
git clone url 非创建者将远程库克隆到本地库
git fetch [别名] [分支] 将远程库某个分支拉取下来
git merge [别名]/[分支] 将远程的分支合并到本地
git poll [别名] [分支] fetch + merge

克隆的效果：

  1. 将远程库下载到本地
  2. 创建远程地址别名
  3. 初始化本地库

邀请别人加入团队：

github项目中——>settings——>Collaborators——>输入被邀请人的GitHub账号——>Copy invite link——>发送给被邀请人

只有成为团队成员才能将本地库推送到远程库

Ⅳ、跨团队协作
  1. 通过fork将别人的远程库fork到自己账号中
  2. 再完成修改后通过Pull Requests——>new pull request——>create pull request，申请将自己的修改用于原项目
