接口定义语言-pb和thrift

接口描述语言，沟通不同语言编写的程序

protobuf内置类型：

类型 范围 bit
short 16
unsigned short 16
Long 32
Unsigned long 32
long long 64
unsigned long long 64
Float 32
Double 64
long double 15bit指数，64bit带符号小数
Char 8
Wchar 从任何宽字符集编码宽字符，如 Unicode 依赖于实现
string ISO Latin-1，除了 ASCII NUL 以外 可变化
Boolean 未指定
Octet 0 - 255 8
Any 自定义数据类型 可变化

消息类型Message的字段：message字段由基本类型组成，可以指定是repeated或者singular（默认）。一个proto文件中可以有多个消息类型，可以在message中使用reserved指定保留字段的编号或者名称。

工作原理：当使用protoc（官方编译器）或kitty（公司框架）编译proto文件时，会生成对应代码。将消息 序列化为输入输出流。

在解析消息时，没有初始化的字段会被设置为默认值，bool为false，string为空字符串，bytes为空字节，数字类型为0等等。

在枚举类型中，如果设置了allow_alias为true，则可以通过指定不同的枚举变量为相同值来设置别名。

map字段不能repeated，key_type不能是枚举类型，内部无序，生成文本格式时，map按键排序，存在多个key时，保存最后的。

如果要将protocal buffer和rpc一起使用，需要在.protocal文件中定义一个RPC服务接口
