常识积累

积累诸如状态码、协议栈这样的常识知识。

状态码

1XX（Infomational）：信息提示性，表示接收的请求正在处理。

名称 解释
100 Continue 客户端用于确认服务器是否能接受实体，尝试发送100 continue的Except，如果可以接受则返回100 continue，否则返回错误码
101 Switching Protocols 服务器正在根据客户端的指令，将协议切换成Upgrade首部所列的协议

说明：1XX信息提示性状态码是在HTTP/1.1版本中新增的，所以无法被HTTP/1.0识别。会出现这样的新增是基于以下的场景：客户端想向服务器发送一个很大的数据报，但不知道服务器是否愿意接受，则可以发送一个请求报文，其中Except：100 continue，由服务器进行决断，如果愿意接受，则返回100 continue，否则考虑返回417 Exceptation Failed。注意这样的过程并不是严格的，客户端可能在收到服务器确认的状态码之前就发送了较大的数据报（考虑网络拥塞），唯一严格的是只能由客户端发起100 continue请求，服务器做出回应，不能由服务器主动发送该响应报文。

2XX（Success）：请求成功，表示请求正常处理完毕。

名称 解释
200 OK 成功，请求被正常处理
201 Created 已创建，请求成功并且服务器创建了新的资源
202 Accepted 已接受，请求被正常接受，不意味着会被正常处理
203 Non-Autoratative-Infomation 非授权信息，服务器成功处理了请求，但返回的信息可能来自另一来源
204 No Content 无内容，服务器正常处理了请求，但没有返回任何内容（如刷新表单页面）
205 Reset Content 重置内容，告知浏览器重置文档视图，清除当前页面中所有Html表单元素
206 Partial Content 部分内容，成功执行了一个部分GET请求

3XX（Rediction）：重定向，表示需要进行附加操作以完成请求。

名称 解释
300 Multiple Choise 多种选择，针对请求，服务器可执行多种操作
301 Moved Permanently 永久性重定向，请求的资源已经永远移动到新位置，会自动将请求转移到新位置（重定向）
302 Found 临时性重定向，表示请求的资源暂时被分配了新的URL，将来的请求还将是老的URL。注：POST请求遇到302会转为GET请求，在HTTP/1.0生效
303 See Other 查看其他位置，告知客户端使用另一个URL获取资源，主要目的是允许POST请求的响应将客户端定位到某个资源上去。HTTP/1.1生效
304 Not Modified 未修改，客户端发送了某个有条件的请求，该条件未能满足，服务端返回该状态码，如客户端想访问修改过的网页，但是网页未被修改
305 Use Proxy 使用代理，告知客户端只能通过代理访问网页
307 Temporary Redirect 临时重定向，对302的一个改进，POST不会变为GET

4XX（Client Error）：客户端错误，表示服务器无法处理请求。

大多数被浏览器解决，常见的只有404

名称 解释
400 Bad Request 错误请求，请求报文中存在语法错误
401 Unauthorized 未授权，请求要求身份验证。对于登录后请求的网页，服务器可能返回此响应
402 Payment Required 保留中
403 Forbidden 禁止，服务器拒绝请求
404 Not Found 未找到，服务器找不到请求的资源
405 Method Not Allowed 方法禁用，禁用请求中指定的方法
406 Not Accepted 不接受，客户端指明自己愿意接收的实体类型，但是服务器中没有相对应的实体
407 proxy Authentication Required 需要代理授权，类似401，指定请求者应当授权使用代理
408 Request TImeout 请求超时，服务器等候请求时发生超时
409 Conflict 冲突，服务器完成请求时发生冲突，需要在响应报文中包含相关信息
410 Gone 已删除，请求的资源已经永久删除
411 Length Required 需要有效长度，服务器只接受含有有效内容长度标头字段的请求
412 Precondition Failed 未满足前提条件，服务器未满足请求者在请求中设置的其中一个请求条件
413 Request Entity Too large 请求实体过大，服务器无法处理过大的请求
414 Request URL Too Long 请求的URI过长，请求的URI过长服务器无法处理
415 Unsupported Media Type 不支持的媒体类型，请求格式不受请求页面的支持
416 Request Range Not Satisfiable 请求范围不符合要求，页面无法提供请求的范围
417 Expectation Failed 未满足期望值，服务器未满足‘Except’请求标头字段的要求

5XX（Server Error）：服务器错误，表示服务器处理请求出错。

名称 解释
500 Internal Server Error 服务器内部错误，无法完成请求
501 Not Implemented 尚未实施，服务器不具备完成功能的请求（如无法识别方法）
502 Bad Gateway 错误网关，服务器作为网关或代理，从上游服务器收到无效响应
503 Service Unavailable 服务不可用，服务器目前无法使用
504 Gateway Timeout 网关超时，服务器作为网关或代理，没有及时从上游服务器收到请求
505 HTTP Version Not Supported HTTP版本不受支持，服务器不支持请求中的HTTP协议版本

注：ping用的协议就是ICMP（字节面试问到）

字节三面问了ping的实现原理……我又懵了，看都看不完整，面试完这就补上。

ping是基于ICMP协议的，由于IP协议是不提供可靠传输的，所以丢包后发送方无法获取通知，针对这个问题，产生了ICMP协议，它主要的功能就是确定数据是否成功发送到了目标位置，如果没有产生了什么错误。

它是这样做的：ICMP报头中有一个属性TYPE，这个属性的不同取值被规定了不同的含义，当发送方发送信息时，TYPE属性规定为8，代表回送请求，如果能成功到达，则会返回数据包，并将ICMP报头的TYPE属性置0。如果没有收到响应数据包或者TYPE属性不为0，则代表目标不可达或其他的一些问题。

将TYPE所有取值记录如下：

TYPE 含义
0 回送应答
3 目标不可达
4 原点抑制
5 重定向或改变路由
8 回送请求
9 路由器公告
10 路由器请求
11 超时
17 地址子网请求
18 地址子网应答

注：traceroute可以用来追踪经过的路由器，其底层原理也是通过ICMP协议
