栈

1. 使用队列实现栈（LC225）

思路：栈的特点是先进后出，队列的特点是先进先出，所以需要将队列中先进的元素放到队列的尾部，为了节省空间考虑，可以使用一个队列完成功能，即：当在队列尾部插入一个元素后，将该元素前面的其他元素依次弹出并插入到尾部。

3. 判断栈的弹出序列是否合法（LC946）

思路：分别用一个栈和一个队列来记录压入和弹出序列，压入时和队列的peek元素对比，如果一样，则说明该元素可以合法弹出，故栈和队列都弹出peek元素，当遍历完压入序列后，如果栈空，说明所有的元素都可以合法弹出，否则就有非法弹出的元素。

问题记录：本来想使用LinkedList作为队列，但是LeetCode提交时挂掉了，死在了一个很长的输入数组中，所以改成使用index记录，更省空间。而且，LinkedList的add()方法是插入尾部，而push()方法是插入头部。顺便对LinkedList集合容器各个方法做一个总结

  1. add / remove
    • add = addLast , 区别是add返回boolean，addLast无返回值。addFirst无返回值，插入头部。
    • remove = removeFirst，无元素throw new NoSuchElementException，有元素返回被删除的元素值
  2. push / pop / poll
    • push = addFirst
    • pop = removeFirst
    • poll = pollFirst 删除头部元素，若无元素返回null，有元素返回该元素的值。pollLast 删除尾部元素

push 和 pop 构成了栈操作，add 和 remove 构成了队列操作，poll在某些不想抛异常的情况下替代remove

堆

预备知识：

*最大（小）二叉堆：最大（小）值先出的完全二叉树，也叫优先队列**

1. 数组中的第K个最大的元素（LC215）

思路：第一反应数组排序，返回length-k下标的数，AC了而且耗时2ms。如果用小顶堆来做也很简单，维护一个K大小的堆，遍历数组，当堆未满时，直接添加，当堆满时，判断该数组元素是否大于堆顶元素，如果是，则替换之，如果不是，继续遍历。该方法时间复杂度按理是O(n)，但实际比数组排序的O(nlogn)更加耗时些，遍历数组是O(n)，所以多出的时间开销应该是用于建立和维护堆，所以当K值较大时这种方法并不合适。

扩展：

从大顶堆的构建中很明显可以看出一个排序算法：堆排序。这个算法的核心思想大概是这样的：一组无序序列进行初始化堆后，堆顶元素一定是最大值，故将该值与堆叶子节点的元素交换位置，交换过位置的元素就是有序序列了。然后重复这个过程，直到所有数组都有序。如果是二叉树实现的堆结构，则可以使用一个队列来层次遍历获得有序数组。

而Java中堆是通过数组实现的，其左右节点对应的值通过数学表达式：2index+1，2\index+2来构建关系，故堆序后只要遍历该数组，就可以得到有序数组。

2. 数据流的中位数(LC703)

思路：要在一个随机存取的数据流中动态的获得中位数，简单的在每次取出中位数的时候对数组排序是不行的，时间复杂度会爆炸，所以要使用一个动态平衡的数据结构，始终维护数组的中位数。一个大顶堆和一个小顶堆就能很巧妙的解决这个问题：大顶堆维护数组的前一半元素，其堆顶元素是前一半元素中最大的，而小顶堆恰恰相反，堆顶元素是后一半元素最小的，所以需要get中位数时，只要对这两个堆顶元素操作即可，并不关系其他元素是否有序。

一般来说，是需要使用大量的if else来判断各种情况的，但是如果牺牲一点点性能，就能使整个代码非常简洁美妙。
