在我看来，动态规划与递归的不同点在于：递归是结果导向的，根据递推式和终止条件，可以很轻松的写出解决问题的代码，至于是否会有大量重复计算，不care。而动态规划则是条件导向的，根据刚开始的条件一步步顺序推导到最后的结果，这个过程中肯定是没有重复计算的，但是代码自然不像递归那样简洁。

面对这个问题时，状态转移方程(1)就是这段

res[0] = Math.max(left[0],left[1]) + Math.max(right[0] ,right[1]);

这句话揭示了当该节点不选时，其取值为左节点的最大值和右节点的最大值中较大的那一个，而某个节点的最大值是该节点选时的最大值和不选时的最大值中较大的那个，有点两重循环的意思。

自然，状态转移方程(2)如下

res[1] = left[0] + right[0] + root.val;

该节点选时，最大值就是这个节点的值加上左节点不选的值再加上右节点不选的值。

两个状态转移方程合起来才是真正的状态转移方程，还是比较难的，反正我是没写出来。

经过打家劫舍的历练，对于动态规划的理解更深了一层（这里记录一下，变形二并没有重复计算，这是一个二叉树，从根节点往左是一条路，往右是另一条截然不同的路，没有任何一个值是重复计算的），总的来说，动态规划就是浪费了一点空间记录了中间过程，获得了巨大的时间提升。

动态规划进阶二: 0-1背包问题

问题引入 : 有一个小偷要偷一些东西，这些东西有两个属性：重量(W)和价值(v),小偷有一个固定容量的背包，怎么才能获得最大价值呢？

很明显，这是一个限定条件下的求最值问题，动态规划，准没错。对于动态规划，找两个东西：状态转移方程和出口条件。

所谓的状态转移方程就是对dp数组的一步步填写，而dp数组的某个值一定取决于已经填好的某个值，所以，定义好dp数组是比较关键的。

在这种题中，有一个类似的模板dp:dp[m][n],其中m为前m个数据，n为限定条件，dp[m][n]为在这个限定条件下所能获得的最大利益。很明显，当数组填充完成，数组的右下角那个数字就是能获得的最大利润。

那么，dp[m][n]的值怎么确定呢？

dp[m][n]一定是两个东西比较之后得到的，一个东西是选，一个东西是不选。

在这道题中，如果选，则dp[m][n] = dp[m-1][n-Wm] + Vm ; 解释如下:当你选择了第m个东西的时候，你获得了这个东西的价值Vm，失去了背包的容量Wm，然后用n-Wm空间的背包去装前m-1个东西，所以dp[m][n] = Vm + dp[m-1][n-Wm]

如果不选，则dp[m][n] = dp[m-1][n], 解释如下: 当你不选第m个东西的时候，你失去了这个东西的价值，但是没有失去背包空间，所以，你可以用n空间的背包去装前m-1个东西，去吧。

那么，这两个选择就一定能获得最大价值吗？当然，这是一个不进行重复计算的递归过程，对于摆在你面前的这个东西，你除了装或者不装，难道还有其他选择吗？你选择了装，有一个已知的结果摆在你的面前，你选择不装，也有一个结果摆在你的面前，这不是很好比较吗？（当然，这个选择也是依赖于之前选择的）所以递归和动态规划，并没有什么本质的不同，动态规划只是小心谨慎的递归，避免进行重复计算而已。代码：
