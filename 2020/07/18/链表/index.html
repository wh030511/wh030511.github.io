3. 相交链表（LC160）

编写一个程序，找到两个单链表相交的起始节点。

思路：一种特别巧妙的方法，用两个指针分别指向两个单链表头，然后依次向前移动，如果两个指针相撞，则相撞点就是相交的起始节点。当指针指向null后，则转向另一个单链表。由于这两个指针最终都会遍历两个链表，所以最后会同时指向null，如果出现这种情况，说明没有相交的点。

说明：

  1. 正确性说明：大多数情况下，两个指针会遍历完两个链表，那么他们走的路程是一样长的，如果存在相交的节点，那么从这个节点往后，两个指针走的路程也是一样长的，简单的数学思想可以看出，如果存在相交节点，那么两个指针会在某一时刻同时指向该相交节点。
  2. 如果恰巧两个链表中相同位置的节点的值相同，并不会造成错误。因为这里使用的是==，比较的是节点的地址，只有当这两个节点是同一个节点时，才会判断相等，既然是同一个节点了，那么一定其后继也相同。
  3. 如果两个链表长度一样长，也不会造成错误。当两个指针都遍历完了自己的链表，同时等于null，其实就可以提前结束循环了，必然不可能存在相交节点。即使两个指针互换了位置，也不过是上一轮检测的简单重复而已。pA变成了pB，pB变成了pA，再次遍历没有意义。

4. 环形链表（LC142）

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

思路：快慢指针可以判断是否有环，如果需要得到环开始的节点，第一反应是使用Set集合，将节点的地址放到集合中，第一次失败的节点就是环开始节点。这个方法可以通过但是时间复杂度很高（Set的创建与数次扩容）。

使用快慢指针也可以找到环开始节点，需要一点数学推导。推导的结果是，如果存在环，从meet位置和head位置以相同的位置出发，相遇的节点就是换节点。

5. 链表划分（LC86）

给定一个链表和一个特定值 x，对链表进行分隔，使得所有小于 x 的节点都在大于或等于 x的节点之前。你应当保留两个分区中每个节点的初始相对位置

思路：遍历链表，将其中元素取出放入动态数组中，遍历数组，找到第一个大于等于x的数，记录下标为flag，从该元素下一个元素开始，每找到一个元素值小于x，则将其插入到flag位置，同时flag自增（为了满足初始相对位置不变），遍历完成后，依次将动态数组中的值赋给单链表。

6. 深拷贝链表（LC138）

**给定一个链表，每个节点包含一个额外增加的随机指针，该指针可以指向链表中的任何节点或空节点。要求返回这个链表的 深拷贝。 **

思路：实际上，我们可以通过序列化和反序列化的手段简单的实现深度拷贝。如果使用算法，也有多种思路。

第一种，通过使用多个Map集合，一个记录原链表的节点和其随机节点下标，一个记录新链表的节点下标。在简单的一次遍历后，我们可以获得没有随机节点的原链表的深度拷贝，然后再次遍历原链表，在遍历的时候同时访问Map，获得该节点对应的随机节点的下标位置。然后新链表的该节点的随机节点也是在另一个Map中相应角标位置的节点，就可知直接指定。

第二种方法更加巧妙些，没有使用多余的数据结构，只是通过不停的改写节点的指向，来达成目的。主要思想就是先将原链表double，将原先的ABC顺序变成AABBCC，然后根据原先的随机节点，拷贝到新链表中，最后断开连接，返回新链表。

第三种就是将链表转为图，通过BFS来拷贝，注意处理死循环的问题。
